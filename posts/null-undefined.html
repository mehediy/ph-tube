Tell us the use cases of null and undefined

null and undefined are two special values in JavaScript, and they have specific use cases:

  1. null:
    • Explicit Absence of Value: null is often used to explicitly indicate that a variable or object property has no value or that it represents the absence of an object.
    • Resetting Values: You can assign null to a variable or property to reset it to an empty or initial state.
    • As a Placeholder: Sometimes, null can be used as a placeholder for a value that will be assigned later. It indicates that a value is expected but hasn't been provided yet.
  2. undefined:
    • Default Value: Variables that are declared but not initialized are automatically assigned the value undefined.
    • Function Parameters: If a function parameter is not provided with a value, it is set to undefined by default.
    • Missing Object Properties: When you try to access a property that doesn't exist on an object, it returns undefined.
    • Return Value of Functions: If a function does not explicitly return a value, it returns undefined.
